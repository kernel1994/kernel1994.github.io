Linux 奇技淫巧之让任务在后台稳定运行

0x00
2017-09-07
Linux

一般我们习惯使用 SSH 登录到远程 Linux 服务器，若提交的是一些耗时的任务，万一中途有个闪失，断线了，那可咋整啊？不要慌张，请立即参见此文章，写得很好，我在此仅总结以及补充额外内容。

nohup

  • nohup COMMAND

setsid

  • setsid COMMAND

disown

  • 用 disown -h jobspec 来使某个作业忽略 HUP 信号
  • 用 disown -ah 来使所有的作业都忽略 HUP 信号
  • 用 disown -rh 来使正在运行的作业忽略 HUP 信号

screen

  • 用 screen -dmS session name 来建立一个处于断开模式下的会话（并指定其会话名）。
  • 用 screen -list 来列出所有会话。
  • 用 screen -r session name 来重新连接指定会话。
  • 用快捷键 CTRL-a d 来暂时断开当前会话。

补充知识

前台/后台任务

在命令的结尾加上 & 可以将任务放入后台执行。此时可以通过 jobs 来查看后台任务列表。

上例中任务 ping www.google.com & 是在后台运行的，如果我们想要其在前台/后台切换，就需要用到 fg (foreground) 和 bg %1 (background) 命令。

此时任务已经在前台运行，若想再将其放入后台，则需要时使用快捷键 Ctrl + Z 来使其挂起在后台，再使用 bg %1 命令将后台挂起的任务在后台继续运行。

可见上例中，前台任务在运行时，我们使用 Ctrl + Z，则任务被挂起在后台，状态为 Stopped，接着使用 bg %1 命令，任务在后台继续运行，状态为 Running。

所以，日常我们使用 Ctrl + Z、fg 以及 bg 就能任意切换前后台任务，非常有意思。

重定向

使用 nohup 命令时，标准输出和标准错误缺省会被重定向到 nohup.out 文件中，使用 >filename 2>&1 可以重定向缺省的输出文件名。

众所周知 echo 的作用是将在屏幕上输出一句话，而此时这句话被重定向到了文件 stroy 中。这是如何办到的呢？这就不得不介绍一下文件描述符（File Descriptor）。

在 Linux 中有三种文件描述符：

  • 0 标准输入（stdin）
  • 1 标准输出（stdout）
  • 2 标准错误（stderr）

所以，其实 $ echo “It was a dark and stormy night. Too dark to write.” > story 是将 stdout 重定向到文件的省略写法，因为 stdout 是默认值。完整写法是 $ echo “It was a dark and stormy night. Too dark to write.” 1> story。

要看懂 2>&1 还需要知道 &1 的作用，这表示引用 stdout 文件描述符（1）的值。所以 2>&1 的意思的将 sdterr 重定向到与 stdout 相同位置。

  2. sdterr 被重定向到 stdout 的重定向的位置，也是文件 proud.log 中
